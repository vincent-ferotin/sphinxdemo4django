Projet Django (sphinxdemo)¶

Un projet Django consiste ici en une instance de sphinxdemo, consituée de sa configuration dans le fichier settings.py, sa base de données associée, etc.

settings.py¶

Configuration d’une instance de projet Django pour l”app hello.

Designating the settings

sphinxdemo.settings.ALLOWED_HOSTS = []¶

Liste blanche d’hôtes autorisés à se connecter au projet.

Laisser vide pour un site publique, sinon expliciter les clients autorisés.

sphinxdemo.settings.BASE_DIR = PosixPath('/home/work/projects/sphinx/sphinxdemo4django/work')¶

Chemin du répertoire racine du dépôt git du présent projet (pathlib.Path).

sphinxdemo.settings.DATABASES = {'default': {'ATOMIC_REQUESTS': False, 'AUTOCOMMIT': True, 'CONN_HEALTH_CHECKS': False, 'CONN_MAX_AGE': 0, 'ENGINE': 'django.db.backends.sqlite3', 'HOST': '', 'NAME': PosixPath('/home/work/projects/sphinx/sphinxdemo4django/work/data/database.sqlite3'), 'OPTIONS': {}, 'PASSWORD': '', 'PORT': '', 'TEST': {'CHARSET': None, 'COLLATION': None, 'MIGRATE': True, 'MIRROR': None, 'NAME': None}, 'TIME_ZONE': None, 'USER': ''}}¶

Configuration de la (ou les) base(s) de données du projet.

Pour cette app Django très simple, il n’y a qu’une seule base de données à configurer, nommée 'default'.

Pour le développement de l”app comme pour un déploiement à petite échelle, une base SQLite suffit amplement, et permet de se dispenser de monter et configurer un serveur de bases de données dédié, comme MariaDB ou PostgreSQL. Le chemin de la base est ici configuré par défaut pour avoir comme répertoire parent DATA_DIR, défini dans ce même fichier settings.py.

Voir aussi

sphinxdemo.settings.DATA_DIR = PosixPath('/home/work/projects/sphinx/sphinxdemo4django/work/data')¶

Chemin du répertoire de données du projet Django (ou instance) (pathlib.Path).

C’est dans ce répertoire que pourrait être stockée par exemple la base SQLite.

sphinxdemo.settings.INSTALLED_APPS = ['hello.apps.HelloConfig', 'django.contrib.auth', 'django.contrib.contenttypes', 'django.contrib.sessions', 'django.contrib.messages', 'django.contrib.staticfiles']¶

Liste des apps Django activées pour ce projet / cette instance (list of apps paths or config).

sphinxdemo.settings.ROOT_URLCONF = 'sphinxdemo.urls'¶

Chemin du module définissant les patrons d’URLs gérés par ce projet / cette instance (str of module path).

La première partie (ici 'sphinxdemo') correspond au package du projet, et à adapter si besoin à la configuration de votre déploiement. La seconde partie (ici 'urls') correspond au module sphinxdemo/urls.py.
